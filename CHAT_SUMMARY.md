## 🎯 项目背景

- **项目名称**：FlowyAIPC（QClaw AI 对话助手）
- **项目性质**：旧 UI 升级改造，需要更新 PRD
- **核心目标**：编写一份**简洁、逻辑闭环、聚焦交互细节**的 PRD，方便开发实现
- **HTML 文件清单**：
  - chat.html（对话首页）
  - agent.html（专家中心）
  - scheduled-tasks.html（定时任务）
  - settings.html（设置）
  - knowledge-base.html（知识库）
  - channels-ui.html（频道）

---

## 🎨 工作风格

1. **多轮讨论为主**：先讨论框架 → 确认细节 → 输出文档
2. **以通用板块为准**：所有具体页面需与通用规范保持一致
3. **表格形式呈现**：按 HTML 文件分大板块，板块内分子模块
4. **术语统一**：用户可见文案用"专家/技能"，文件路径保留英文
5. **遇到迟疑就问**：不擅作主张，遇到不明确的问题就提出来
6. **中等详细度**：聚焦核心交互逻辑和边界条件，不写视觉细节

## 📊 已达成共识的关键规则

### 1. 数据规格

| 字段 | 限制 |
|------|------|
| 单条消息 | ≤20000 字符（≈10000 字中文） |
| 专家名称 | ≤50 字符 |
| 专家描述 | ≤1500 字符 |
| 搜索关键词 | ≤50 字符（不实时显示） |
| chat 侧边栏专家数 | ≤50 个 |
| 单个专家子对话数 | ≤30 条 |
| 官方 Agent 总数 | ≤100 个 |
| 技能勾选上限 | ≤15 个 |
| 技能卡片每行 | 3 个（通用） |
| 首页对话卡片每行 | 4 个（chat 3.5） |

### 2. 弹窗规则

- **5 种位置分类**：模态弹窗 / 顶部 Toast / 底部 Toast / 行内提示 / 侧边抽屉
- **层级规则**：系统级弹窗可与 UI 弹窗共存，UI 自定义弹窗同时只能打开一个
- **时长**：成功提示 2 秒自动关闭，错误/确认弹窗手动关闭

### 3. 字数提示规则

- **表单字段**（专家名称、描述）：实时显示字数
- **长文本**（对话消息、搜索）：不实时显示，超出截断并提示

### 4. 搜索规则

- 全局模糊搜索
- 实时搜索
- **隐藏非匹配项**（不高亮）

### 5. 交互逻辑

- **专家名称不可编辑**（主专家）
- **新建专家在 agent.html**，跳转路径 chat → agent → 创建 → chat
- **数据同步**：chat 与 agent.html 共享专家数据
- **模型**：默认 Auto，未选择模型不置灰
- **附件**：预解析未完成发送按钮置灰，上传失败显示刷新按钮

## 💬 关键决策点（重启对话时可快速引用）

1. **术语**：用户可见用"专家/技能"，文件名保留英文（agent.html/skill.html）
2. **新建专家入口**：只在 agent.html 4.2，chat 板块 3.3 不重复
3. **首页对话卡片** ≠ **技能卡片**：首页对话卡片每行 4 个，技能卡片每行 3 个（通用）
4. **专家名称不可编辑**（没有可编辑 icon 的需求）
5. **子对话容器**（左侧）：固定高度≈10 条，超过 3 条显示"查看更多"，鼠标滚动
6. **对话消息区**（右侧）：无限滚动，可向上滑到首次对话

## 🎯 模式总结（快速开启新对话时引用）

> 这是一个 UI 升级改造的 PRD 项目。需要按照 HTML 文件分大板块，每个板块内用表格分子模块（序号、功能点、规则描述、边界条件、异常处理）。用户偏好"中等详细度"，聚焦交互细节和边界条件，不写视觉细节和动画。讨论时遇到迟疑就提问，多轮确认后再写入文档。术语上展示给用户用"专家/技能"，文件路径保留英文。所有具体页面的规则必须与通用规范保持一致。
