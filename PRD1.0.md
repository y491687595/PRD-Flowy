## 1. 产品概述

### 1.1 核心使用场景

FlowyAIPC 是一款 AI 对话助手产品，提供多专家管理、智能对话、技能扩展等功能，帮助用户高效完成各类任务。

### 1.2 用户角色定义

| 角色   | 描述               | 主要功能需求       |
| ---- | ---------------- | ------------ |
| 普通用户 | 使用 AI 对话功能完成日常任务 | 对话、专家管理、技能使用 |

## 2. 通用规范

### 2.1 数据规格定义

#### 2.1.1 字符限制规范

| 字段     | 限制值                  | 交互行为                                          |
| ------ | -------------------- | --------------------------------------------- |
| 单条消息   | ≤20000 字符（≈10000字中文） | 超出截断并提示"超出字数限制（20000字符）"，**用户单次发送的对话消息总字数限制** |
| 专家名称   | ≤50 字符               | 超出禁止输入，实时字数提示                                 |
| 专家描述   | ≤1500 字符             | 超出禁止输入，实时字数提示                                 |
| 搜索关键词  | ≤50 字符               | 超出禁止输入，不实时显示字数（搜索框仅显示已输入字符，不显示剩余字数）           |
| 技能名称   | ≤40 字符               | 超出禁止输入，不实时显示字数                                |
| 技能描述   | ≤1600 字符             | 超出禁止输入，不实时显示字数                                |
| 知识库名称  | ≤100 字符              | 超出禁止输入，不实时显示字数                                |
| 定时任务名称 | ≤100 字符              | 超出禁止输入，不实时显示字数                                |
| 定时任务指令 | ≤1500 字符             | 超出禁止输入，不实时显示字数（已从原 §2.1.2 提升至此，行为更明确）              |
| 反馈描述   | ≤1000 字符             | 超出禁止输入；右下角实时显示 `当前字符数 / 1000`；见 §7.5.1 #1        |
| 反馈邮箱   | ≤100 字符              | 超出禁止输入；弱校验 `xxx@xxx.xxx` 格式；见 §7.5.1 #3             |
| channel 账号命名 | ≤20 字符         | 超出禁止输入；见 §9.1.1 #7                              |

#### 2.1.2 数量上限规范

| 字段            | 限制值                                               | 交互行为                                                                 |
| ------------- | ------------------------------------------------- | -------------------------------------------------------------------- |
| chat 侧边栏专家总数  | ≤50 个                                             | 超限提示"已达最大专家数量，请删除不再使用的 专家后再试"                                        |
| 单个专家子对话数      | ≤30 条                                             | 超限提示"该专家对话数量已达上限（30个），请删除部分历史对话"；A + B 共用此额度，见 §3.3 #7          |
| agent板块官方专家总数 | ≤100 个                                            | 由官方自行控制上架数量                                                          |
| 技能卡片每行显示      | 3 个                                               | 响应式自适应（通用规范）                                                         |
| 首页对话卡片每行显示    | 4 个                                               | 响应式自适应（chat 板块独立规格）                                                  |
| 新建专家技能勾选上限    | ≤15 个                                             | 超限提示"最多勾选15个技能"                                                      |
| 已安装技能总数       | ≤50 个                                             | 超限提示"已达最大技能数量（50个），请删除不再使用的技能后再试"                                    |
| 技能文件上传大小      | ≤10 M                                             | 超出提示"文件太大啦，请压缩到 10M 以内再上传"                                           |
| 知识库总数         | 不限                                                | -                                                                    |
| 单知识库文件数量上限    | ≤50 个                                             | 超限提示"该知识库已达最大文件数（50个），请删除部分文件后再上传"                                   |
| 知识库单文件大小上限    | ≤20 M                                             | 超出提示"文件太大啦，请压缩到 20M 以内再上传"                                           |
| 知识库支持文件类型     | .md / .pdf / .docx / .pptx / .xlsx / .html / .htm | 不在范围内提示"暂不支持该文件类型，请上传 .md / .pdf / .docx / .pptx / .xlsx / .html 格式" |
| 定时任务指令字符      | ≤1500 字符                                          | 超出禁止输入，不实时显示字数（详细规范见 §2.1.1 "定时任务指令" 行）                              |
| 定时任务总数        | ≤50 个                                             | 超限提示"已达最大任务数（50个），请删除部分任务后再创建"                                       |
| 定时任务间隔范围      | 1 - 1440 分钟（24 小时）                                | 超出禁止输入；空时保存按钮置灰                                                      |
| 定时单次预约最大提前    | ≤60 天                                             | 过去日期/时间不可选                                                           |
| 保护目录数         | ≤50 个                                             | 超限提示"已达最大保护目录数（50个）"，添加按钮置灰；见 §7.6.3 #1                          |
| 单频道账号数        | ≤3 个                                              | 超限提示"已达最大账号数（3个）"，添加按钮置灰；见 §9.1.1 #2                           |
| 累计账号总数        | ≤24 个（8 频道 × 3 账号）                                | -                                                                    |

#### 2.1.3 字段格式规范

| 字段类型  | 格式要求                                       |
| ----- | ------------------------------------------ |
| 专家名称  | 支持中英文、数字、特殊符号，不允许纯空格                       |
| 专家描述  | 支持中英文、数字、特殊符号、换行                           |
| 搜索关键词 | 支持中英文、数字                                   |
| 任务名称  | 支持中英文、数字、特殊符号，不允许纯空格                       |
| 任务指令  | 支持中英文、数字、特殊符号、换行                           |
| 时间格式  | HH:mm，24 小时制                               |
| 日期格式  | YYYY-MM-DD                                 |
| 星期格式  | 周一/周二/周三/周四/周五/周六/周日（内部索引 0-6，周一为 0，周日为 6） |
| 间隔单位  | 整数，单位为分钟                                   |
| 任务名重复 | 允许同名的多个任务（名称不作为唯一标识）                       |
| 反馈描述  | 支持中英文、数字、特殊符号、换行                           |
| 反馈邮箱  | 弱校验 `xxx@xxx.xxx` 正则；填错可继续提交但不进 send 队列      |
| 账号 ID | 系统生成 `ad0d9f778c63-im-bot` 格式，不可编辑          |
| channel 账号命名 | 支持中英文、数字、特殊符号，不允许纯空格                  |
| 积分数值  | 整数（积分余额）/ 小数点后 2 位（积分消耗，如 `-2.25`）       |
| B 类子对话标题 | `[频道名] 发送者：消息前 12 字`               |
| channel 状态值 | 4 种：在线 / 离线 / 连接中 / 异常                    |

### 2.3 交互状态与反馈规范

#### 2.3.1 按钮状态定义

| 状态  | 视觉表现                   | 交互行为                  |
| --- | ---------------------- | --------------------- |
| 可用态 | 正常颜色、正常 cursor         | 可点击，点击后执行操作           |
| 禁用态 | 置灰、cursor: not-allowed | 不可点击，悬停显示禁用原因 Tooltip |
| 悬停态 | 背景色加深/阴影               | 鼠标指针变化                |
| 点击态 | 按下效果（transform 缩放）     | 防重复点击，执行中显示 loading   |

#### 2.3.2 典型按钮置灰场景

| 场景    | 置灰规则                                  |
| ----- | ------------------------------------- |
| 输入框为空 | 发送按钮置灰禁用                              |
| 未选择模型 | 默认选中auto，无需置灰                         |
| 积分不足  | 发送按钮置灰 + 提示"积分不足，点这里获取积分"，点击后跳转至官网付费页 |
| 文件上传中 | 发送按钮置灰禁用                              |

#### 2.3.3 弹窗交互规范

**弹窗展示位置分类：**

| 弹窗类型     | 展示位置    | 适用场景             |
| -------- | ------- | ---------------- |
| 模态弹窗     | 屏幕中央    | 创建成功、错误确认等需要用户决策 |
| 顶部 Toast | 顶部居中/右上 | 操作成功、轻量提示        |
| 底部 Toast | 底部居中    | 移动端为主            |
| 行内提示     | 操作行附近   | 删除/编辑确认等局部操作     |
| 侧边抽屉     | 右侧滑出    | 详情查看、设置面板等       |

**弹窗交互规则：**

| 弹窗类型      | 自动关闭时长 | 关闭方式    | 备注             |
| --------- | ------ | ------- | -------------- |
| 操作成功提示    | 2 秒    | 可手动关闭   | 顶部 Toast       |
| 操作失败/错误提示 | 不自动关闭  | 必须手动关闭  | 模态弹窗           |
| 二次确认弹窗    | 不自动关闭  | 取消/确认按钮 | 模态弹窗           |
| 轻提示 Toast | 2 秒    | 可手动滑动关闭 | 顶部 Toast，统一为2秒 |

**弹窗层级规则：**

- 系统级弹窗（如文件选择、浏览器原生弹窗）可与 UI 弹窗共存
- UI 自定义弹窗同时只能打开一个，避免多层嵌套

#### 2.3.4 提示文案规范

| 规范项  | 要求                          |
| ---- | --------------------------- |
| 按钮文字 | 无歧义，明确表达操作结果（如"删除"vs"确认删除"） |
| 跳转说明 | 明确跳转到哪个页面或模块                |
| 操作后果 | 删除等不可逆操作需明确告知后果             |

#### 2.3.5 表单验证规范

| 验证方式      | 适用范围                |
| --------- | ------------------- |
| **实时验证**  | 专家名称（必填、字数限制）、字符数统计 |
| **提交时验证** | 表单完整性检查、格式校验        |
| **双重验证**  | 关键字段采用实时+提交双重验证     |

### 3.4 对话消息区

> **B 类子对话消息操作规则**（2026-07-15 配套 §9 channels 章节新增）：
> - A 类：所有消息操作（复制/点赞/点踩/重新生成）均可正常操作
> - B 类：复制/点赞/点踩可操作；**重新生成不可操作**（B 类消息来自 channel 端用户，不应被 AI 重新生成）
> - B 类滚动行为与 A 类相同

| 序号 | 功能点   | 规则描述             | 边界条件 | 异常处理       |
| -- | ----- | ---------------- | ---- | ---------- |
| 1  | 用户消息  | 右侧展示用户消息         | -    | -          |
| 2  | AI 消息 | 左侧展示 AI 回复       | -    | 网络异常显示重试按钮 |
| 3  | 消息操作  | 复制、点赞、点踩、重新生成、朗读、分享（共 6 个） | A 类 6 个全可用；B 类仅前 3 个可用（重新生成/朗读/分享 禁用） | -          |
| 4  | 对话滚动  | 无限滚动，可向上滑动查看历史对话 | A / B 类行为相同 | 直至首次对话内容   |

## 9. channels.html - 频道模块

> 频道是 FlowyAIPC 与外部 IM 平台（微信/飞书/QQ/Telegram/Discord/WhatsApp/钉钉/WeCom）通信的入口；基于 OpenClaw 协议实现
> 核心概念：
> - **1 个 channel 账号 = 1 个 B 类 chat 子对话**（飞书、飞书（1）各对应一个 B 子对话）；每个 channel 账号独立绑 1 个 agent
> - **同 channel 类型的多个账号必须共享同一 agent**（与 N2.2 "一个对话只能绑一个频道一个 agent" 一致）
> 与其他模块联动：
> - §3 chat：channel 消息实时同步为 chat 中的 B 类子对话（详见 §9.7）
> - §6 定时任务：定时任务勾选远控通道后，执行结果走对应 channel B 子对话；不勾选走 main agent（见 §6.8.7）
> - §7.3.1 设置：设置页微信（登录账号）与频道微信（机器人绑定通信）功能独立
> 数据存储：channel 配置 + 账号 ID + token 等存本地浏览器（localStorage）
> 本章节不展开的内容（已开发）：OpenClaw 插件内部通信机制、token 加密存储细节、具体鉴权 UI 实现

### 9.5 8 频道连接差异

> 每个频道的差异点列在"差异点"列；通用步骤见 §9.4

#### 9.5.1 微信

| 项      | 描述                                       |
| ------ | ---------------------------------------- |
| 平台     | 微信                                       |
| 鉴权方式   | 二维码扫码（用户用微信扫一扫）                          |
| 鉴权流程   | 弹出二维码 → 用户扫 → 手机端确认 → 客户端轮询（3 秒/次）→ 确认后获取 token |
| 二维码有效期 | 5 分钟（过期后需刷新）                            |
| 频率限制   | 单客户端 5 分钟内最多重试 3 次                      |
| 差异点    | 唯一需要"扫一扫"操作                            |
| 异常     | 二维码过期 / 用户取消 / 轮询超时 → 重试按钮回到步骤 2          |

#### 9.5.2 飞书

| 项     | 描述                                              |
| ----- | ----------------------------------------------- |
| 平台    | 飞书 / Lark                                       |
| 鉴权方式  | OAuth 2.0 授权                                   |
| 鉴权流程  | 跳转飞书登录 → 用户登录 + 授权 → 飞书回调 → 获取 access_token      |
| Token | access_token + refresh_token                  |
| 刷新    | access_token 过期前 5 分钟自动刷新                    |
| 差异点   | 唯一需要"跳转外部页面"的频道                            |
| 异常    | 用户取消授权 / 回调超时 / refresh_token 失效 → 重连重新走流程       |

#### 9.5.3 QQ

| 项     | 描述                                              |
| ----- | ----------------------------------------------- |
| 平台    | QQ 机器人                                          |
| 鉴权方式  | Bot Token（QQ 开放平台获取）                          |
| 鉴权流程  | 用户在 QQ 开放平台获取 Bot Token → 客户端输入 → 验证 → 保存      |
| Token 格式 | 32 位字符串                                         |
| 差异点   | 唯一"纯 Token"输入，无扫码无 OAuth                    |
| 异常    | Token 无效 / 已过期 / 格式错误 → 提示具体错误，重连可重新输入         |

#### 9.5.4 Telegram

| 项     | 描述                                                       |
| ----- | -------------------------------------------------------- |
| 平台    | Telegram                                                |
| 鉴权方式  | Bot Token（@BotFather 获取）                                |
| 鉴权流程  | 用户在 Telegram 联系 @BotFather 创建机器人 → 客户端输入 Token → 调用 getMe 验证 → 保存 |
| Token 格式 | `<bot_id>:<token>`                                      |
| 差异点   | 与 QQ 类似，但走 Telegram 协议                              |
| 异常    | Token 无效 / 机器人已删除 / 网络异常 → 重连按钮                    |

#### 9.5.5 Discord

| 项     | 描述                                                  |
| ----- | --------------------------------------------------- |
| 平台    | Discord                                            |

| 鉴权流程  | 开发者门户创建机器人 → 获取 Token → 客户端输入 → 验证 → 保存        |
| Token 格式 | 字符串                                                 |
| 差异点   | 与 Telegram 类似                                      |
| 异常    | Token 无效 / 权限不足 / 网络异常 → 重连                       |

#### 9.5.6 WhatsApp

| 项      | 描述                                          |
| ------ | ------------------------------------------- |
| 平台     | WhatsApp                                   |
| 鉴权方式   | 二维码扫码（无需手机号）                                |
| 鉴权流程   | 弹出二维码 → 用户用 WhatsApp 扫 → 关联设备 → 客户端获取凭证    |
| 二维码有效期 | 2 分钟                                       |
| 差异点    | 扫码 + 关联设备流程；无 Token 概念                    |
| 异常     | 二维码过期 / 关联失败 / 网络异常 → 重连                   |

#### 9.5.7 钉钉

#### 9.5.8 WeCom（企业微信）

| 差异点   | 与微信不同，是"企业微信机器人"，需管理员权限                |
| 异常    | 二维码过期 / 非管理员扫码 / 网络异常 → 重连                  |

### 9.6 失败与异常处理

| 序号 | 场景       | 处理                                                                                          |
| -- | -------- | ------------------------------------------------------------------------------------------- |
| 1  | 连接失败     | Toast 显示具体错误（token 无效 / 二维码过期 / 权限不足等）+ 弹窗内显示「重试」按钮                                         |
| 2  | 连接中断（已连接后） | 状态点变黄 + 尝试自动重连（3 次，每次间隔 10 秒）                                                                   |
| 3  | 自动重连失败   | 状态点变灰 + Toast "channel 已离线，请检查网络或重新连接"                                                            |
| 4  | 离线超过 5 分钟 | 状态点保持灰 + 不再自动重连（避免持续打扰用户）                                                                     |
| 5  | 重新上线     | 状态点变绿 + 离线期间缓存的 channel 消息（如平台支持）补传到 chat 对应 B 子对话                                          |
| 6  | 账号被平台封禁  | 状态点变红 + Toast "账号已被平台封禁，请查看帮助" + 引导用户去 §7.5 反馈或重连                                          |
| 7  | Token 失效（连接后） | 尝试自动 refresh；refresh 失败 → 状态点变灰 + Toast "Token 已失效，请重新连接"                                      |
| 8  | channel 关闭（用户主动） | 状态点变灰 + 对应 B 子对话禁用（不接新消息），**保留历史**                                                          |
| 9  | agent 卸载（B 子对话绑定的） | 兜底回 main agent + Toast "agent 已解绑，现由 main agent 处理 channel 消息"                                |
| 10 | main agent 卸载 | main agent 不可卸载；B 子对话保持当前绑定状态不变                                                            |

### 9.7 chat 子对话联动（B 类机制）

> 与 §3 chat 模块联动：每个 channel 账号对应一个 B 类子对话

| 序号 | 规则         | 描述                                                                                                          |
| -- | ---------- | ----------------------------------------------------------------------------------------------------------- |
| 1  | B 子对话来源     | **每个 channel 账号 = 1 个 B 类子对话**（飞书、飞书（1）各 1 个）；B 类与 A 类（客户端输入创建的子对话）独立                 |
| 2  | B 子对话生成时机   | 第一次成功连接 channel 账号时自动创建；删除 channel 账号时 B 子对话**禁用**（保留历史，不删）                                  |
| 3  | B 子对话归属专家   | 绑定的 agent（默认 main agent，可下拉切换）                                                                       |
| 4  | B 子对话标题     | `[频道名] 发送者：消息前 N 字`（N 取 12 字）                                                                     |
| 5  | B 子对话 icon  | 绿色 `#01C260` 对话气泡 + 圆点（参考 chat.html 历史区 "QClaw的邮件服务..."前的图标风格）；A 类为灰色对话气泡                  |
| 6  | B 子对话只读     | 用户无法在 B 子对话中以客户端方式输入；输入区替换为提示："该对话来自 [微信/飞书等]，请在 channel 端继续交流" + 隐藏发送按钮              |
| 7  | B 子对话实时同步   | channel 端每发一条消息 → 实时同步到 B 子对话（≤3 秒延迟）                                                            |
| 8  | B 子对话上下文     | channel 端每次新消息都会传到 B 子对话上下文，AI 持续监听并自动回复                                                       |
| 9  | B 子对话与定时任务  | 定时任务勾选 channel 后，执行结果走对应 B 子对话；不勾选走 main agent 主专家会话                                          |
| 10 | B 子对话与 §3.3 #7 限制 | A 类与 B 类**共用 30 条/专家**；B 类占用会减少 A 类可用额度（详见 §3.3 #7 修订）                                      |
| 11 | B 子对话编辑/删除  | B 类**不可编辑标题、不可删除**（避免破坏 channel 历史）；仅可通过 §9.3.3 删除 channel 账号来"禁用"它                       |
| 12 | B 子对话激活规则   | 用户点击 B 子对话项 → 右侧消息区切换到该 B 子对话（只读模式），输入区替换为只读提示                                            |
| 13 | 离线消息补传     | 客户端重新上线后，channel 离线期间的缓存消息（如平台支持）按时间顺序追加到 B 子对话                                          |
