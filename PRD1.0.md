## 1. 产品概述

### 1.1 核心使用场景

FlowyAIPC 是一款 AI 对话助手产品，提供多专家管理、智能对话、技能扩展等功能，帮助用户高效完成各类任务。

### 1.2 用户角色定义

| 角色   | 描述               | 主要功能需求       |
| ---- | ---------------- | ------------ |
| 普通用户 | 使用 AI 对话功能完成日常任务 | 对话、专家管理、技能使用 |

## 2. 通用规范

### 2.1 数据规格定义

#### 2.1.1 字符限制规范

| 字段    | 限制值                  | 交互行为                                          |
| ----- | -------------------- | --------------------------------------------- |
| 单条消息  | ≤20000 字符（≈10000字中文） | 超出截断并提示"超出字数限制（20000字符）"，**用户单次发送的对话消息总字数限制**    |
| 专家名称  | ≤50 字符               | 超出禁止输入，实时字数提示                                |
| 专家描述  | ≤1500 字符             | 超出禁止输入，实时字数提示                                |
| 搜索关键词 | ≤50 字符               | 超出禁止输入，不实时显示字数（搜索框仅显示已输入字符，不显示剩余字数）              |

#### 2.1.2 数量上限规范

| 字段            | 限制值    | 交互行为                             |
| ------------- | ------ | -------------------------------- |
| chat 侧边栏专家总数  | ≤50 个  | 超限提示"已达最大专家数量，请删除不再使用的 专家后再试"    |
| 单个专家子对话数      | ≤30 条  | 超限提示"该专家对话数量已达上限（30个），请删除部分历史对话" |
| agent板块官方专家总数 | ≤100 个 | 由官方自行控制上架数量                      |
| 技能卡片每行显示      | 3 个    | 响应式自适应                           |
| 新建专家技能勾选上限    | ≤15 个  | 超限提示"最多勾选15个技能"                  |

#### 2.1.3 字段格式规范

| 字段类型  | 格式要求                 |
| ----- | -------------------- |
| 专家名称  | 支持中英文、数字、特殊符号，不允许纯空格 |
| 专家描述  | 支持中英文、数字、特殊符号、换行     |
| 搜索关键词 | 支持中英文、数字             |

***

### 2.2 搜索与筛选规范

| 规范项        | 规则描述              |
| ---------- | ----------------- |
| **搜索范围**   | 全局搜索（所有专家）    |
| **搜索方式**   | 模糊搜索（包含关键词即显示）    |
| **搜索实时性**  | 实时搜索，输入即搜索        |
| **搜索结果展示** | 隐藏非匹配项，仅显示匹配项；无匹配项时显示空状态提示 |

***

### 2.3 交互状态与反馈规范

#### 2.3.1 按钮状态定义

| 状态  | 视觉表现                   | 交互行为                  |
| --- | ---------------------- | --------------------- |
| 可用态 | 正常颜色、正常 cursor         | 可点击，点击后执行操作           |
| 禁用态 | 置灰、cursor: not-allowed | 不可点击，悬停显示禁用原因 Tooltip |
| 悬停态 | 背景色加深/阴影               | 鼠标指针变化                |
| 点击态 | 按下效果（transform 缩放）     | 防重复点击，执行中显示 loading   |

#### 2.3.2 典型按钮置灰场景

| 场景    | 置灰规则                                  |
| ----- | ------------------------------------- |
| 输入框为空 | 发送按钮置灰禁用                              |
| 未选择模型 | 默认选中auto，无需置灰                         |
| 积分不足  | 发送按钮置灰 + 提示"积分不足，点这里获取积分"，点击后跳转至官网付费页 |
| 文件上传中 | 发送按钮置灰禁用                              |

#### 2.3.3 弹窗交互规范

**弹窗展示位置分类：**

| 弹窗类型   | 展示位置       | 适用场景              |
| ------ | ---------- | ----------------- |
| 模态弹窗   | 屏幕中央       | 创建成功、错误确认等需要用户决策 |
| 顶部 Toast | 顶部居中/右上    | 操作成功、轻量提示          |
| 底部 Toast | 底部居中       | 移动端为主             |
| 行内提示   | 操作行附近      | 删除/编辑确认等局部操作       |
| 侧边抽屉   | 右侧滑出       | 详情查看、设置面板等        |

**弹窗交互规则：**

| 弹窗类型      | 自动关闭时长 | 关闭方式    | 备注              |
| --------- | ------ | ------- | --------------- |
| 操作成功提示    | 2 秒    | 可手动关闭   | 顶部 Toast        |
| 操作失败/错误提示 | 不自动关闭  | 必须手动关闭  | 模态弹窗            |
| 二次确认弹窗    | 不自动关闭  | 取消/确认按钮 | 模态弹窗            |
| 轻提示 Toast | 2 秒    | 可手动滑动关闭 | 顶部 Toast，统一为2秒 |

**弹窗层级规则：**
- 系统级弹窗（如文件选择、浏览器原生弹窗）可与 UI 弹窗共存
- UI 自定义弹窗同时只能打开一个，避免多层嵌套

#### 2.3.4 提示文案规范

| 规范项  | 要求                          |
| ---- | --------------------------- |
| 按钮文字 | 无歧义，明确表达操作结果（如"删除"vs"确认删除"） |
| 跳转说明 | 明确跳转到哪个页面或模块                |
| 操作后果 | 删除等不可逆操作需明确告知后果             |

#### 2.3.5 表单验证规范

| 验证方式      | 适用范围                |
| --------- | ------------------- |
| **实时验证**  | 专家名称（必填、字数限制）、字符数统计 |
| **提交时验证** | 表单完整性检查、格式校验        |
| **双重验证**  | 关键字段采用实时+提交双重验证     |

### 2.4 导航与页面规范

| 规范项        | 规则描述             |
| ---------- | ---------------- |
| **页面跳转规则** | 点击导航项跳转，保留原页面状态  |
| **页面状态保留** | 返回后恢复到跳转前的位置和状态  |
| **返回恢复机制** | 保持滚动位置、筛选条件、展开状态 |

***

### 2.5 反馈与提示规范

#### 2.5.1 加载状态设计

| 场景     | 加载形式                    | 备注            |
| ------ | ----------------------- | ------------- |
| 页面初始加载 | 骨架屏                     | 保持布局稳定        |
| 数据请求   | Loading 动画 + 文字"加载中..." | 超时（10秒）显示重试按钮 |

#### 2.5.2 空状态设计

| 空状态场景   | 占位提示文案               |
| ------- | -------------------- |
| 无 专家    | "暂无专家，点击上方按钮创建第一个专家" |
| 无消息     | "开始对话吧，向我发送消息"       |
| 无搜索结果   | "未找到匹配的专家，请尝试其他关键词"  |
| 已安装内无技能 | "暂无技能，前往技能市场安装"      |

#### 2.5.3 字数实时提示

| 适用字段        | 提示方式                                |
| ----------- | ----------------------------------- |
| 专家名称、描述    | 输入时实时显示已输入字符数，达到上限时提示"已达上限（XX/XX）" |
| 对话消息、搜索关键词 | **不实时显示**字数提示，超出限制时截断并提示"超出字数限制"   |

#### 2.5.4 操作反馈

| 操作类型  | 反馈形式                     |
| ----- | ------------------------ |
| 操作成功  | Toast 提示"操作成功"，2秒后消失     |
| 操作失败  | Toast 提示错误原因，不自动消失，需手动关闭 |
| 不可逆操作 | 二次确认弹窗，明确告知后果            |

***

### 2.6 数据存储规范

| 规范项        | 规则描述                       |
| ---------- | -------------------------- |
| **存储策略**   | 本地存储（LocalStorage）+ 云端同步   |
| **离线可用性**  | 基础功能（查看历史对话）支持离线使用，发送消息需联网 |
| **数据同步机制** | 联网时自动同步，冲突时以云端数据为准         |
